Phase 01

Local Setup Protocols

Choose your operating system below. Each protocol provides step-by-step instructions to deploy Ollama on your local machine with full offline capability.

WARNING: Ensure your system meets minimum requirements — 8GB RAM recommended, GPU with CUDA/Metal support for optimal performance.

STEP 01

Download & Install Ollama

The fastest way to install Ollama on Linux is using the official install script. This script handles all dependencies automatically.

bash 
$ curl -fsSL https://ollama.com/install.sh | sh
STEP 02

Verify Installation

Confirm Ollama is running by checking the version and service status.

bash 
$ ollama --version
ollama version is 0.4.7

$ systemctl status ollama
● ollama.service - Ollama Service
   Loaded: loaded (/etc/systemd/system/ollama.service)
   Active: active (running)
STEP 03

Pull Your First Model

Download a model to start running inference locally. Llama3.2 is recommended for beginners.

bash 
$ ollama pull llama3.2
pulling manifest
pulling 4a39a... 100%
verifying sha256 digest
writing manifest
success

# Run the model interactively
$ ollama run llama3.2
STEP 04

Configure GPU Acceleration (Optional)

For NVIDIA GPUs, ensure CUDA drivers are installed. Ollama auto-detects and uses GPU acceleration.

bash 
# Check NVIDIA drivers
$ nvidia-smi

# Set custom environment variables if needed
$ export OLLAMA_HOST=0.0.0.0:11434
$ export OLLAMA_ORIGINS=*

# Restart service
$ systemctl restart ollama
STEP 01

Download Ollama for macOS

Download the official macOS application bundle or use Homebrew for a CLI installation.

bash 
# Option A: Using Homebrew (recommended)
$ brew install ollama

# Option B: Direct download
$ curl -o ollama-darwin.zip https://ollama.com/download/Ollama-darwin.zip
$ unzip ollama-darwin.zip
$ mv Ollama.app /Applications/
STEP 02

Start the Ollama Service

If installed via Homebrew, start the service. If using the app, simply launch it from Applications.

bash 
# Start Ollama as a background service
$ brew services start ollama

# Verify it's running
$ ollama list
NAME      ID      SIZE      MODIFIED
STEP 03

Pull & Run a Model

macOS supports Metal GPU acceleration natively. Pull and run any model.

bash 
$ ollama pull llama3.2

# Run interactively
$ ollama run llama3.2
>>> Send a message (/? for help)

# Or use the REST API
$ curl http://localhost:11434/api/generate -d '{
  "model": "llama3.2",
  "prompt": "Hello from AiCrime",
  "stream": false
}'
STEP 04

Apple Silicon Optimization

On M1/M2/M3 Macs, Ollama automatically uses the Neural Engine and GPU. Verify Metal acceleration is active.

bash 
# Check GPU utilization during inference
$ ollama run llama3.2 --verbose

# Look for "gpu" in the load info
load model
  total duration:       1.2s
  load duration:        800ms
  prompt eval count:    1
  eval duration:        200ms
  GPU:                  Metal (Apple M2)
STEP 01

Download Ollama for Windows

Download the Windows installer from the official site or use winget via PowerShell.

powershell 
# Option A: Using winget (recommended)
PS> winget install --id Ollama.Ollama

# Option B: Direct installer download
# Visit: https://ollama.com/download
# Download OllamaSetup.exe and run it

# Option C: Scoop package manager
PS> scoop install ollama
STEP 02

Verify Installation

Open PowerShell or Command Prompt and verify Ollama is accessible.

powershell 
PS> ollama --version
ollama version is 0.4.7

PS> ollama serve
time=2025-01-15T... level=INFO source=routes.go:...
msg="Listening on 127.0.0.1:11434"
STEP 03

Pull & Run Models

Use the same commands across all platforms. Pull a model and start chatting.

powershell 
# Pull a model
PS> ollama pull llama3.2

# Run the model
PS> ollama run llama3.2
>>> Send a message (/? for help)
>>> Who are you?
I am a large language model trained by Meta.

# List downloaded models
PS> ollama list
NAME            ID           SIZE    MODIFIED
llama3.2        a694fb51a842  2.0GB   2 minutes ago
STEP 04

GPU Setup (NVIDIA)

For GPU acceleration on Windows, install the latest NVIDIA drivers. Ollama auto-detects CUDA-capable GPUs.

powershell 
# Verify CUDA GPU detection
PS> ollama run llama3.2 --verbose

# Check for GPU in output
GPU:    NVIDIA GeForce RTX 4070
Layers: 33/33 (100% GPU offload)

# Set custom port if needed
PS> setx OLLAMA_HOST "0.0.0.0:11434"
# Restart Ollama after setting env vars

100% Offline

All inference happens locally. No data leaves your machine. No API keys required. No telemetry.

GPU Accelerated

NVIDIA CUDA, AMD ROCm, and Apple Metal are all supported. Automatic GPU offloading.

REST API

Built-in OpenAI-compatible API endpoint at localhost:11434. Drop-in replacement for cloud APIs.

Model Library

Access hundreds of open-weight models: Llama, Mistral, Phi, Gemma, Qwen, and many more.

Phase 02

Developer Interface

Ollama exposes a REST API at localhost:11434. Integrate with any language or framework. Full OpenAI compatibility mode available.

API Endpoints
POST /api/generate Generate completion
POST /api/chat Chat completion
POST /api/embeddings Get embeddings
GET /api/tags List models
POST /api/pull Pull model
DELETE /api/delete Delete model
quick_start.py
python 
import requests

# Generate a response
response = requests.post(
    "http://localhost:11434/api/generate",
    json={
        "model": "llama3.2",
        "prompt": "Explain quantum computing",
        "stream": False
    }
)

print(response.json()["response"])
