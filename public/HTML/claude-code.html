Phase 01

Quick Start Protocol

Follow these steps to connect Claude Code to your local Ollama instance. Total setup time: ~15 minutes. Requires Node.js 18+ and 8GB+ RAM.

PREREQUISITES: Ollama installed & running, Node.js 18+, and a model pulled (e.g., qwen2.5-coder:14b).

STEP 01

Verify Ollama is Running

Ensure Ollama service is active and your model is loaded.

bash
$ ollama serve
$ ollama list
NAME                    ID              SIZE
qwen2.5-coder:14b       a1b2c3d4        9.2 GB
STEP 02

Install LiteLLM Proxy

LiteLLM translates Claude Code's requests to Ollama's API.

bash
$ pip install "litellm[proxy]"
STEP 03

Configure LiteLLM

Create a config file mapping Claude names to local models.

yaml
# ~/litellm_config.yaml
model_list:
  - model_name: claude-3-5-sonnet-20241022
    litellm_params:
      model: ollama/qwen2.5-coder:14b
      api_base: http://localhost:11434
general_settings:
  master_key: sk-local-dev-key
STEP 04

Start Proxy & Configure Env

Launch LiteLLM and point Claude Code to it.

bash
$ litellm --config ~/litellm_config.yaml --port 8000
$ export ANTHROPIC_BASE_URL=http://localhost:8000
$ export ANTHROPIC_API_KEY=sk-local-dev-key
STEP 01

Verify Ollama & Install Proxy

Run these in PowerShell.

powershell
PS> ollama serve
PS> pip install "litellm[proxy]"
STEP 02

Set Environment Variables

Make variables persistent for Claude Code.

powershell
PS> [System.Environment]::SetEnvironmentVariable('ANTHROPIC_BASE_URL', 'http://localhost:8000', 'User')
PS> [System.Environment]::SetEnvironmentVariable('ANTHROPIC_API_KEY', 'sk-local-dev-key', 'User')
STEP 03

Start Proxy & Test

Launch LiteLLM, then run Claude Code in your project folder.

powershell
PS> litellm --config C:\Users\You\litellm_config.yaml --port 8000
PS> cd C:\path\to\project
PS> claude

Phase 02

Troubleshooting Matrix

ISSUE 01

Connection Refused

Verify both services are running and env vars match.

bash
$ curl http://localhost:11434/api/tags
$ curl http://localhost:8000/health
$ echo $ANTHROPIC_BASE_URL
ISSUE 02

Tool Use Fails

Use instruct/chat variants, not base models.

bash
$ ollama list
# Ensure model name does NOT end with "-base"
